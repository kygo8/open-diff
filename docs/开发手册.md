# Open Diff 开发手册

> 技术路线：Tauri 2 + Rust Core + Vue 3 + Naive UI + 自研 Diff 视图  
> 目标：构建一个面向普通用户的跨平台本地差异比较与合并工具。

---

## 1. 架构原则

Open Diff 采用“轻 UI 壳 + 强 Rust Core”的架构：

- Vue 3 负责界面组织、状态管理和用户交互。
- Naive UI 负责通用控件，不负责核心对比视图。
- Rust Core 负责 diff、merge、扫描、同步、解析、报告和脚本等重逻辑。
- Tauri command 只做薄封装，不承载业务规则。

推荐分层：

```text
Vue UI
├─ layouts      应用外壳：菜单栏、标签页、状态栏
├─ views        页面：Home、Text Compare、Settings
├─ components   通用组件和自研 diff 组件
├─ stores       Pinia 状态
├─ api          Tauri command 封装
└─ styles       主题 token 和桌面密度

Rust
├─ shared-types 前后端 DTO
├─ diff-core    文本 diff 和差异模型
├─ folder-core  文件夹扫描与比较
├─ session-core 会话和设置
└─ app commands Tauri command 薄封装
```

---

## 2. Naive UI 使用边界

Naive UI 用于应用壳和标准控件：

- 按钮、输入框、单选、多选。
- 弹窗、抽屉、通知、消息。
- 设置页表单。
- 菜单、下拉、进度条。
- 简单列表和管理型面板。

Naive UI 不用于核心 Diff 视图：

- Text Diff 主对比面板。
- Folder Diff 文件树表。
- Table Diff 大数据网格。
- Hex Viewer。
- Image Diff 画布。
- Diff Minimap。

原因：

- 核心对比区域需要虚拟滚动、稳定布局、细粒度高亮、键盘导航和大文件优化。
- 通用 UI 表格容易带来后台管理风格，不适合专业桌面工具。
- 自研视图更容易控制性能和交互细节。

---

## 3. 自研 Diff 视图规范

所有 Diff 视图都应满足：

- 固定行高或可预测行高。
- 支持虚拟滚动。
- 左右内容同步滚动。
- 差异状态使用统一颜色 token。
- 不因长行或长路径撑坏布局。
- 数据模型与渲染组件分离。
- 大数据只渲染可见区域。

统一差异状态：

```ts
type DiffLineKind = 'equal' | 'added' | 'deleted' | 'modified'
```

后续扩展状态：

- `conflict`
- `unimportant`
- `left-only`
- `right-only`
- `error`

---

## 4. Rust Core 规范

Rust Core 按能力拆分 crate。每个 crate 必须可以脱离 Tauri 单独测试。

规则：

- Core crate 不依赖 Tauri。
- Core crate 接收明确输入，返回结构化输出。
- 错误必须带上下文。
- 大文件处理必须支持分块或增量模型。
- 文件夹扫描、同步、报告生成必须可取消。
- 同步操作必须先生成 plan，再执行。

当前最小 crate：

- `shared-types`：共享 DTO。
- `diff-core`：文本 diff 最小实现。

后续建议新增：

- `folder-core`
- `sync-core`
- `merge-core`
- `table-core`
- `image-core`
- `hex-core`
- `vfs-core`
- `report-core`
- `script-core`
- `session-core`

## 5. Tauri Command 设计规范

Tauri command 只负责：

- 接收前端参数。
- 调用 Rust Core。
- 转换错误。
- 返回 DTO。

不要在 command 中写复杂业务逻辑。

推荐形式：

```rust
#[tauri::command]
pub fn diff_text(left: String, right: String) -> TextDiffResponse {
    let request = TextDiffRequest { left, right };
    diff_core::diff_text(&request)
}
```

大任务推荐形式：

```text
start_folder_scan(options) -> job_id
job_progress event
get_job_result(job_id) -> result page
cancel_job(job_id)
```

## 6. 前端状态管理规范

使用 Pinia，按领域拆分：

- `settings`：主题、字体、显示偏好。
- `tabs`：打开的标签页和脏状态。
- `sessions`：保存的对比会话。
- `jobs`：后台任务状态。
- `profiles`：远程连接配置。

状态原则：

- UI 临时状态留在组件内。
- 跨页面状态进入 store。
- Rust Core 产生的大结果不要全部塞进 store。
- 大型 diff 结果应按分页或范围读取。

---

## 7. 主题与视觉规范

普通用户优先：

- 默认深色主题。
- 提供浅色主题。
- 使用紧凑桌面密度。
- 控件圆角不超过 8px。
- 差异颜色必须柔和但清晰。
- 颜色状态必须配合符号或文字，不只依赖颜色。

核心 token：

- `--diff-added-bg`
- `--diff-added-fg`
- `--diff-deleted-bg`
- `--diff-deleted-fg`
- `--diff-modified-bg`
- `--diff-modified-fg`
- `--diff-gutter-bg`

```powershell

```

```powershell
pnpm dev
```

```powershell

```

```powershell
pnpm typecheck
```

```powershell
pnpm test
```

```powershell

cargo test
```

```powershell
pnpm build
pnpm tauri:build
```

## 9. 开发路线

### 阶段 1：基础闭环

- Vue + Naive UI 应用壳。
- Tauri command 调用。
- Rust `diff-core`。
- Text Compare 最小闭环。
- 设置页和主题切换。

### 阶段 2：可用文本对比

- Myers/Patience/Histogram 算法。
- 行内字符高亮。
- 虚拟滚动。
- 搜索和跳转。
- 文件打开和编码处理。

### 阶段 3：文件夹对比与同步

- 递归扫描。
- 差异状态。
- 过滤。
- 同步 plan。
- 执行前确认。

### 阶段 4：专业视图

- Table Diff。
- Hex Diff。
- Image Diff。
- Text Merge。
- Folder Merge。

### 阶段 5：自动化与集成

- CLI。
- 脚本。
- 报告。
- 右键菜单。
- Git difftool / mergetool。
- 远程 VFS。

## 10. 质量要求

每个核心能力必须包含：

- Rust 单元测试。
- 前端组件测试。
- 错误状态展示。
- 大输入性能策略。
- 文档说明。

完成前必须运行：

```powershell
pnpm typecheck
pnpm test
Set-Location src-tauri
cargo test
cargo check
```
