# Beyond Compare 1:1 复刻需求规格说明书

> 文档版本：v1.0  
> 编写日期：2026-06-26  
> 需求基线：Beyond Compare 5.2.2 build 32209，发布日期 2026-05-20  
> 适用项目：OpenDiff  
> 目标：以功能、交互、工作流和集成能力为主线，定义 Beyond Compare 的 1:1 复刻需求。本文不要求复刻商标、品牌资产、专有图标、授权系统或闭源实现细节。

---

## 1. 背景与目标

### 1.1 产品定位

Beyond Compare 是一款跨平台数据对比与合并工具，核心价值是让用户在文件、文件夹、表格、图片、二进制、注册表、远程资源、归档包和版本控制工作流中快速发现差异、理解差异并安全地同步或合并结果。

OpenDiff 的 1:1 复刻目标是：

- 覆盖 Beyond Compare 5 Pro 的主要用户可见能力。
- 保持同类工作流的操作路径、信息架构和快捷键体验尽量一致。
- 优先复刻对开发者、运维、内容编辑、数据处理用户最常用的对比、过滤、同步、合并、报告、脚本和外部集成能力。
- 在开源实现中避免复制 Beyond Compare 的商标、品牌图形和专有资源，但功能语义、视图结构和工作流要尽量对齐。

### 1.2 成功标准

达到 1:1 复刻的最低成功标准：

- 用户能从首页创建所有主要 Session 类型，并能保存、搜索、锁定、复用、自动保存和共享 Session。
- 文本、文件夹、表格、十六进制、图片、注册表、媒体、补丁、文本编辑等视图均可按 Beyond Compare 的常见任务完成对比。
- 文件夹同步、文本合并、文件夹合并支持预览、冲突标记、批量执行和结果保存。
- 支持本地文件、网络路径、FTP/SFTP/WebDAV/S3/Dropbox/OneDrive/Subversion、归档文件和快照等数据来源。
- 支持命令行、脚本、右键菜单、版本控制 difftool/mergetool、报告导出、设置导入导出。
- 关键操作可键盘驱动，并支持自定义快捷键、字体、颜色、主题和文件格式规则。

### 1.3 版本与版本形态

本需求以 Beyond Compare 5.2.2 为基线。产品形态按功能完整度分层：

- 标准能力：常规文件、文件夹、表格、图片、二进制对比，同步、报告、脚本、命令行、会话管理。
- Pro 能力：文本合并、文件夹合并、SFTP/FTPS、云资源、Subversion、注册表对比、文本替换忽略、文件夹对齐覆盖、部分源控集成。
- 平台特定能力：Windows 右键菜单、Windows 注册表对比、Windows 便携设备、macOS Apple Silicon、Linux 包与桌面集成。

## 2. 用户与使用场景

### 2.1 目标用户

- 开发者：比较源码、补丁、分支变更、合并冲突、二进制文件、生成审查报告。
- 运维和发布人员：同步目录、比对部署包、比较远程服务器、生成差异报告。
- 数据分析人员：比较 CSV、TSV、Excel、HTML 表格，按主键、容差和列规则定位差异。
- 内容编辑人员：比较 Word、PDF、RTF 转换后的文本，忽略格式或不重要差异。
- 系统管理员：比较注册表、配置目录、归档包、版本信息、文件属性和权限。

### 2.2 核心场景

- 快速拖放两个文件或文件夹，自动选择合适视图并展示差异。
- 从首页创建一个文件夹对比 Session，配置过滤规则后保存，下次一键恢复。
- 比较两个代码文件，忽略空白、注释、大小写或指定正则片段，只查看重要差异。
- 对三方文本冲突进行合并，自动合并非冲突部分，手动处理冲突部分并保存输出。
- 比较生产目录和发布包，预览同步动作后执行 Update/Mirror。
- 比较两个 Excel 工作簿的多个 Sheet，按关键列对齐并高亮单元格差异。
- 对图片进行像素对比，支持缩放、旋转、翻转、容差和替换颜色忽略。
- 通过命令行或脚本在 CI/发布流程中生成文件夹或文本差异报告。

## 3. 信息架构与通用概念

### 3.1 Session

Session 是一个对比或合并任务的保存单元。每个 Session 必须包含：

- Session 类型。
- 左侧、右侧、中心或输出路径，按类型决定数量。
- 视图布局、过滤、比较规则、显示规则、文件格式规则引用。
- 描述、名称、保存位置、锁定状态。
- 最近使用时间、自动保存状态、是否为共享 Session。

### 3.2 View

View 是打开后的工作视图，可以是首页、文件夹对比、文本对比、表格对比、文本合并等。每个 View 在窗口或标签页中呈现。

### 3.3 Child Session

从文件夹对比中打开单个文件对比时形成 Child Session。Child Session 需要知道父文件夹 Session，以支持“打开下一个差异文件”“复制并跳到下一个差异”等操作。

### 3.4 File Format

File Format 用于决定：

- 某扩展名或文件模式打开时使用哪种视图。
- 比较前是否需要转换。
- 文本语法、语法元素、大小写规则、制表符、行权重。
- 文件夹规则比较时如何判断重要差异。

### 3.5 Difference 状态

所有视图应统一使用以下差异状态：

- Same：两边一致。
- Different：存在重要差异。
- Unimportant：仅存在不重要差异。
- Orphan Left：仅左侧存在。
- Orphan Right：仅右侧存在。
- Conflict：三方合并中左右均修改同一段。
- Unknown：尚未扫描或无法确定。
- Error：读取、解析、权限、网络或转换失败。

## 4. 首页与 Session 管理需求

### HOME-001 首页入口

首页必须在新窗口或新标签打开时展示，并提供以下入口：

- 新建 Folder Compare。
- 新建 Folder Merge。
- 新建 Folder Sync。
- 新建 Text Compare。
- 新建 Text Merge。
- 新建 Table Compare。
- 新建 Hex Compare。
- 新建 Picture Compare。
- 新建 Registry Compare。
- 新建 Text Edit。
- 打开 Text Patch。
- 打开其他视图类型，如 Media Compare、Version Compare。

验收标准：

- 用户可以从首页直接启动任意受支持 Session 类型。
- 首页支持拖放文件或文件夹；拖到具体类型入口时按该类型打开。
- 直接拖放两个文件或文件夹时，应自动选择最合适的 Session 类型。

### HOME-002 Saved Sessions

系统必须支持命名保存 Session：

- 保存当前 Session。
- 另存为新 Session。
- 保存到 Session 文件夹。
- 重命名、删除、复制、移动。
- 锁定 Session，避免误改。
- 修改已保存 Session 的主要参数后，关闭时提示保存。

验收标准：

- 命名 Session 可在首页树中展示。
- 锁定 Session 不能直接覆盖保存，必须先解锁或另存。
- Session 修改后能检测脏状态。

### HOME-003 Auto-Saved Sessions

系统必须自动保存最近 Session：

- 关闭 Session 后可在首页恢复。
- 自动保存数量可配置。
- 用户可清理自动保存历史。

验收标准：

- 异常退出后能恢复最近工作。
- 自动保存 Session 与命名 Session 在 UI 上明确区分。

### HOME-004 Session 搜索与管理面板

首页必须提供：

- Session 名称搜索。
- 按类型、最近使用、文件夹分组。
- 管理面板展示 Session 参数。
- 支持编辑默认 Session 设置。

验收标准：

- 搜索输入后实时过滤 Session 树。
- 对某个 Session 类型编辑默认设置后，新建同类型 Session 自动继承。

### HOME-005 Shared Sessions

系统必须支持共享 Session 文件：

- 导出包含 Session 的设置包。
- 配置共享 Session 文件路径。
- 共享 Session 在 Session 树中以只读分支展示。

验收标准：

- 共享 Session 可打开但不能在本地直接修改。
- 本地修改共享 Session 时必须另存为个人 Session。

### HOME-006 Workspaces

系统必须支持保存工作区：

- 保存所有打开窗口、标签页、Session、布局和尺寸。
- 从命令行或首页恢复命名 Workspace。

验收标准：

- 关闭再打开 Workspace 后，窗口、标签页和 Session 组合恢复一致。

## 5. 通用视图与窗口需求

### UI-001 多窗口与标签页

必须支持：

- 多窗口。
- 单窗口多标签页。
- 新建标签、关闭标签、移动标签。
- 可配置单标签时是否隐藏标签栏。
- Home View、Session View 可在同一标签中互相替换。

### UI-002 布局模式

文件类视图必须支持：

- 左右并排布局。
- 上下布局。
- 同步滚动。
- 分隔条拖拽。
- 适合窄屏的布局折叠策略。

### UI-003 差异导航

所有差异视图必须支持：

- 下一个差异。
- 上一个差异。
- 下一个冲突。
- 上一个冲突。
- 从父文件夹 Session 打开下一个差异文件。
- 复制文件并打开下一个差异文件。

### UI-004 显示过滤

所有适用视图必须支持：

- Show All。
- Show Differences。
- Show Same。
- Show None。
- Show Context，并可配置上下文行数。
- Ignore Unimportant Differences。
- 临时 Suppress Filters。

### UI-005 书签

文本、表格、二进制等行式视图必须支持：

- 设置 0 到 9 编号书签。
- 跳转到指定书签。
- 清除当前书签。
- 清除全部书签。

### UI-006 字体、颜色与主题

系统必须支持：

- Light Mode。
- Dark Mode。
- 跟随系统主题。
- 自定义文件视图字体。
- 放大、缩小、重置字号。
- 自定义差异颜色、背景色、语法高亮色。
- 高 DPI 和每显示器缩放。

### UI-007 快捷键与命令可见性

系统必须支持：

- 内置快捷键。
- 自定义快捷键。
- 菜单和工具栏命令可隐藏或显示。
- 命令面板或命令列表可搜索执行。

### UI-008 状态栏与统计

每个 Session 视图必须提供：

- 比较状态。
- 差异数量。
- 重要差异数量。
- 不重要差异数量。
- 选择项统计。
- 当前编码、换行符、文件格式、过滤状态。
- 错误和警告入口。

## 6. 通用文件对比能力

### FILE-001 自动视图选择

打开一对文件时，系统必须根据 File Format 选择视图：

- 文本类进入 Text Compare。
- CSV/TSV/Excel/HTML Table 进入 Table Compare。
- 图片进入 Picture Compare。
- 二进制进入 Hex Compare。
- .diff/.patch 进入 Text Patch。
- .reg 或注册表资源进入 Registry Compare。
- 媒体标签进入 Media Compare。

### FILE-002 文件内容来源

必须支持：

- 本地文件。
- 剪贴板内容。
- 标准输入。
- 归档包内文件。
- 网络共享路径。
- 远程 Profile 文件。
- 从父 Folder Session 打开的文件。

### FILE-003 文件转换

必须支持比较前转换：

- Word、PDF、RTF 自动转换为文本后比较。
- HTML 可作为文本或格式化 HTML 视图。
- 可为文件格式配置外部转换器。
- 转换错误必须显示为可诊断错误。

### FILE-004 保存与备份

文件编辑类视图必须支持：

- 保存左侧或右侧文件。
- 另存为。
- 保存到远程 Profile。
- 保存前自动备份。
- 只读文件提示。
- 外部修改检测和重新加载提示。

### FILE-005 报告

所有主要对比视图必须支持：

- 打印报告。
- HTML 报告。
- 文本报告。
- 可配置报告布局、列、上下文、仅差异、包含相同项。
- 文件夹报告可链接到单个文件对比报告。

## 7. Text Compare 需求

### TEXT-001 基础对比视图

Text Compare 必须支持两个文本文件的视觉对比：

- 左右或上下布局。
- 行级差异。
- 行内字符级差异。
- 同步滚动。
- 缩略差异导航条。
- 可显示行号。
- 可显示空格和制表符。
- 可切换语法高亮。
- 可开启 Word Wrap。

### TEXT-002 编辑能力

Text Compare 必须支持：

- 左右任一侧内联编辑。
- 动态重新比较。
- 撤销和重做。
- 重新加载文件。
- 在保留编辑内容的情况下重新比较。
- 自动缩进。
- 删除缩进。
- 查找时默认当前词。

### TEXT-003 差异操作

必须支持：

- 将左侧当前差异复制到右侧。
- 将右侧当前差异复制到左侧。
- 复制整段差异。
- 复制选中内容。
- 复制当前文件到另一侧。
- 复制后跳到下一个差异。
- 对选中行或当前段落标记 Ignored。

### TEXT-004 重要性规则

必须支持将以下差异定义为不重要：

- 注释。
- 空白。
- 分隔字符串。
- 正则表达式匹配文本。
- 字符大小写。
- 页眉页脚。
- 指定列范围。
- 换行符风格。
- 手动忽略的选择。
- 文本替换和重命名标识符。

验收标准：

- 重要差异与不重要差异必须使用不同状态和颜色。
- 开启 Ignore Unimportant Differences 后，仅不重要差异应视为相同。

### TEXT-005 对齐能力

必须支持：

- 自动对齐相似行。
- 多种对齐风格。
- 可选择禁止差异对齐。
- 手动 Align With。
- 手动 Break Alignment。
- 三方合并中支持多行手动对齐。

### TEXT-006 搜索与替换

必须支持：

- Find。
- Find Next / Previous。
- Replace。
- 正则搜索。
- 区分大小写。
- 全词匹配。
- 在左侧、右侧或全部面板搜索。

### TEXT-007 详情面板

必须支持：

- Text Details。
- Hex Details。
- Alignment Details。
- Ruler。
- Webpage/HTML 预览面板。

### TEXT-008 Text Patch

必须支持：

- 打开 .diff/.patch 文件。
- 以补丁文件重建侧边对比。
- 从任意视图拖入 .diff/.patch 时打开 Text Patch。
- 补丁视图支持差异导航、搜索、复制、报告。

## 8. Text Merge 需求

### TMERGE-001 三方合并模型

Text Merge 必须支持：

- Left。
- Right。
- Center/Base。
- Output。

验收标准：

- 三个输入文件加一个输出文件时直接进入合并视图。
- 未显式提供输出文件时，可在保存时指定。

### TMERGE-002 自动合并

必须支持：

- 自动合并非冲突修改。
- 标记冲突区段。
- Favor Left。
- Favor Right。
- 只读输入。
- 输出面板编辑。

### TMERGE-003 冲突处理

必须支持：

- 接受左侧。
- 接受右侧。
- 接受中心。
- 手工编辑输出。
- 对整段或选择范围执行合并动作。
- 跳转下一个/上一个冲突。

### TMERGE-004 命令行自动合并

必须支持命令行自动合并：

- 无冲突时自动写入输出并退出。
- 有冲突时打开 UI 或返回非零状态，取决于参数。

## 9. Folder Compare 需求

### FOLDER-001 基础视图

Folder Compare 必须支持两个文件夹结构侧边对比：

- 类文件管理器树表视图。
- 展开和折叠文件夹。
- 打开全部子文件夹。
- 关闭全部子文件夹。
- 自动扫描子文件夹。
- 后台扫描。
- 支持本地、网络、远程、归档和快照作为任一侧。

### FOLDER-002 显示列

必须支持展示和配置列：

- 名称。
- 大小。
- 修改时间。
- 类型。
- 属性。
- CRC。
- 版本信息。
- Unix owner/group/permissions。
- Unix 文件类型。
- Windows 附加属性，如 symbolic link、temporary、offline。

### FOLDER-003 比较准则

必须支持：

- 按修改时间。
- 按大小。
- 按文件名大小写。
- 按 DOS/Windows 属性。
- 按 Unix 属性、权限、owner/group。
- 按内容 CRC32。
- 按字节比较。
- 按文件格式规则比较。
- 按 exe/dll 版本信息比较。

### FOLDER-004 过滤

必须支持三类过滤：

- File Filters：按文件/文件夹名称、通配符、路径、属性包含或排除。
- Display Filters：按相同、差异、孤儿、新旧、错误等状态显示或隐藏。
- Folder Display Filters：控制文件夹何时出现。

验收标准：

- 文件操作默认尊重当前过滤规则。
- 用户可在操作确认中选择包含隐藏项。
- Suppress Filters 后被过滤项临时显示，并以特殊颜色标记。

### FOLDER-005 文件夹操作

必须支持对选中项执行：

- Open。
- Open With。
- Quick Compare。
- Compare To。
- Align With。
- Compare Contents。
- Copy to Left/Right。
- Copy to Folder。
- Move to Left/Right。
- Move to Folder。
- Exchange。
- Delete。
- Rename。
- Change Attributes。
- Touch。
- Exclude。
- New Folder。
- Copy Filename。
- Ignore selected differences。
- Refresh Selection。

### FOLDER-006 内容比较

Compare Contents 必须支持：

- CRC。
- Binary。
- Rules-based。

验收标准：

- 可对选中项执行内容扫描。
- 扫描结果能更新文件夹差异状态。
- Rules-based 比较应结合 File Format 判断重要差异。

### FOLDER-007 同步命令

Folder Compare 内必须提供快速同步：

- Update Right：复制左侧较新项和左侧孤儿到右侧。
- Update Left：复制右侧较新项和右侧孤儿到左侧。
- Update Both：双向复制较新项和孤儿。
- Mirror to Right：右侧变成左侧镜像，替换差异、删除右侧孤儿、复制左侧孤儿。
- Mirror to Left：左侧变成右侧镜像，替换差异、删除左侧孤儿、复制右侧孤儿。

### FOLDER-008 对齐能力

必须支持：

- 同名文件自动对齐。
- 不同扩展名文件对齐。
- 通配符对齐覆盖，如 `Apples*` 对 `Bananas*`。
- Unicode normalization 差异对齐。
- 忽略文件夹结构进行对比。

### FOLDER-009 报告

必须支持：

- Summary 报告。
- Side-by-side 报告。
- XML/文本/HTML 输出。
- 选择列。
- 只报告差异或包含相同项。
- HTML 文件夹报告链接到单个文件报告。

## 25. 非功能需求

### NFR-001 性能

- 大文件加载必须渐进展示，避免 UI 长时间无响应。
- 文件夹扫描必须后台化，可取消。
- 多个文件操作可后台并发执行。
- 二进制比较必须按块读取，避免一次性加载超大文件。
- 远程目录应缓存元数据，并允许刷新。

### NFR-002 稳定性

- 文件读写、网络传输、转换、脚本执行失败不得导致应用崩溃。
- 崩溃后可恢复自动保存 Session。
- 长时间同步任务必须有日志和可恢复线索。

### NFR-003 安全

- 删除、覆盖、镜像同步必须二次确认。
- 远程凭据必须安全存储。
- 导出 Profile 时应可选择是否包含密码。
- silent 脚本不得绕过文件系统权限。
- 运行外部转换器或外部应用时必须有明确来源和错误隔离。

### NFR-004 可访问性

- 主要功能可键盘完成。
- 颜色状态必须配合图标或文字提示。
- 高对比度主题可用。
- 字体大小可调整。

### NFR-005 可观测性

- 同步、脚本、远程传输、转换和报告生成必须有日志。
- 错误应包含路径、操作、原因、建议处理方式。
